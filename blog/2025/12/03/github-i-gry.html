Github i gry

Czy Github nadaje się do tworzenia strony internetowej o tematyce gier RPG? No jasne, że tak. Jest to świetny serwis gdzie możemy wrzucać pliki związane z różną tematyką. Ja wrzucam te związane z grami RPG. A czemu akurat Github? Github to hosting, repozytorium. Nie ma dziś zbyt dużego wyboru w kwestii hostingów, a sam serwis mi bardzo przypadł do gustu. Można go łączyć z GIT-em, a można samemu wrzucać pliki na serwer.

Jak to przystało na system kontroli wersji Github posiada możliwość gromadzenia plików w wielu wersjach. Czyli mam plik blog.html i tam aktualizuję listę wpisów na bloga. Wrzucam więc plik po aktualizacji na serwer, a Github tworzy sam nową wersję pliku. W każdej chwili mogę sobie prześledzić poprzednie wersje pliku blog.html żeby sobie przeanalizować jego strukturę i inne parametry. Tak więc jest to narzędzie w pełni profesjonalne.

Czyli Github służy mi też jako zrzut informacji na temat gier RPG. Stworzę sobie jakieś pliki, a Github mi to wszystko archiwizuje, gromadzi z automatu. No dobra, a czemu Github, a nie np. Blogspot, Vercel, Cloudflare, itd.? Mi z Githubem współpracuje się znakomicie. Na Cloudflare wrzucałem pliki z Githuba z repozytorium, na Vercela też, na Render tak samo. Nie zawsze to mi działało tak płynnie w sensie no jest subdomena, ale pliki i tak mam na Githubie. Cloudflare co prawda daje statystyki oraz własne rozwiązanie zwane CLI, ale bez Githuba guzik tam zrobię. Blogspot jest mega spoko, ale nie mam możliwości konfiguracji szablonu. Jest więc narzędziem mocno okrojonym względem tego co daje mi Github.

Na Github można sobie zamieszczać pliki bez problemu, też o tematyce gier RPG. Jeszcze korzystałem z Glitch czy coś takiego, spoko, ale tam też po pewnym czasie nawet nie mogłem dodawać wpisów. Od nie i tyle. Na Github wymagana jest pełna znajomość tego co się robi czyli pełna struktura, tekst. Bez tego strona o tematyce gier RPG raczej nie powstanie. Jest do wyboru HTML i CSS oraz MD, ja wybrałem to pierwsze. Sluga zrobiłem w ten sposób, że mam blog/2025/12/03/plik.html. Jest to przykładowy link do wpisu o grach RPG. Jakbym dodał pliki do samego katalogu blog to mogło by dojść do takiej sytuacji gdzie dodaję ten sam plik o grach w sensie taki sam slug przez co podmieniłbym plik o grach który już istniał i bym go utracił, a tutaj tego problemu właściwie nie ma.
