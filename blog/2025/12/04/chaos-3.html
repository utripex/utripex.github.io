3 część z serii Chaos. Pierwsza znajduje się na Chaos >>

A czy taniec jest umieszczony w grach RPG? No nie za bardzo. Staraliśmy się pewnie nieco przeforsować taniec, ale dziś to tak nie wygląda. Nie mniej i ten element jest bardzo ważny i można go do gier dodać. Nie mniej dziś eksplorujemy teren. Gry RPG to dziś odwiedzanie różnych zakątków mapy aż po sam ich koniec. W Gothicu 2 bodajże jak się wyszło poza mapę to zaraz był na nią powrót. Jak idziemy w grach to w pewnym momencie mamy jakieś bariery w sensie dalej już nie idziemy. Po prostu, można zawsze dorobić mapę żeby była większa i żeby było jeszcze więcej przestrzeni do zwiedzania. Nowe obiekty, domy, postacie, questy, eq. Tworzenie gildii, grup. I coś tam się rozkręca nie ma co. Możliwość rozwoju gry jest więc ciekawą opcją na to aby była jeszcze potężniejsza, silniejsza, lepsza. Wdrażanie graczy w szczegóły implementacji gry RPG staje się zwiększeniem możliwości strukturalnych całego teamu tworzącego gry. Sztuczna inteligencja czy programiści do tworzenia gry? Zdecydowanie programiści. SI w ogóle nie rozumie programowania, a programista chociaż stara się zrozumieć. Być może w przyszłości SI będzie realnym wsparciem dla programistów gier RPG, ale dziś nie ma co na to liczyć.

Zacząłem tworzyć grę Amri. W grze mamy budżet i myślimy na co go wydać. Grę piszę w Scali, to mi najbardziej odpowiada, ale na forku pro. Normalnie Scala pro nie jest.

// Masz budżet i na co go wydać?
// Gra Amri

object Main {
def main(args: Array[String]) = {
println("Gra Amri")

var budzet = new Budzet
}
}

class Budzet {
private var value: Float = 0

def get() = {
value
}

def set(value: Float) = {
this.value = value
}

def add(value: Float) = {
this.value += value
}

def minus(value: Float) = {
this.value -= value
}
}
